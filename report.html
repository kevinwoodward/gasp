Accurately Sampled Graph Visualizations of IMDb Data

Kevin Woodward, keawoodw@ucsc.edu

CMPS161, Alex Pang, Winter 2017


Project Information

This project is a graph-based intuitive interface for exploring connections between a sample of actors and films defined by the user. This tool was then used to determine what quality or qualities are best to sample root actors by for a diverse graph, as well as how to branch to new actors and films.
User guide:
Fill the text field out with tags of actors, separated by commas. A tag is the unique ID in each IMDb actor URL which follows /name/. For example, the URL for Jeff Goldblum is http://www.imdb.com/name/nm0000156/, so the corresponding tag is nm0000156. From here, a graph can be generated or the graph options can be modified.

Options

-The first dropdown menu sets the preference on how the search algorithm will choose a film from the filmography of a given actor. The options are newer, older, randomly.
-The second dropdown menu sets the preference on how the search algorithm will choose an actor from a given film. The options are high billing, low billing, and randomly.
-The third dropdown menu is the layout of the graph as it is generated. The options are concentric (by degree outward), circular, and breadthfirst.
-The fourth dropdown menu determines the action when a node is clicked. The options are opening the related IMDb page in a new tab, dragging the nodes, or performing one of three algorithms with the selected node as the source. Edges will always open the related film page.
-The animate graph checkbox can be unchecked for higher performance with larger graph sizes.
-The branching edges input field allows a modification of how many films (edges) will be added per node.
-The graph scale factor input is the limiting factor for the size of the graph, determined by a combination of search depth from the root nodes as well as total node count. Note that this scales noticeably to time and performance.
-Recenter simply centers and fits the graph to the viewbox.

Details:
The general flow of the program is to take the source actors and recursively expand on each through a long series of GETs and callbacks while periodically cross checking nodes for edges and fulfilling the options selected. All of the information gathered during a generation is stored in a single object managed by Cytoscape.js.

This project utilizes several libraries. Cytoscape.js is used for graph manipulation, organization, visualization, and graph based algorithms. RESTful JSON API for IMDb is used for requesting info about an actor or film using IMDb tags. In turn, this uses the IMDb alternative interfaces as reference. jQuery is used for ajax GETs as well as simple DOM manipulation. W3 templates are used for this document.
